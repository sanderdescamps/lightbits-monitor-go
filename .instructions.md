# Coding instructions for lightbits-monitor-go

## Documentation sync — required after every change

After **any** change to `main.go`, always update **both** documentation files in the same response:

- **`README.md`** — user-facing documentation.
  - Keep the **Flags table** in sync with all `f.XxxVarP` / `f.XxxVar` calls in `main()`.
  - Update **Examples** when new flags or behaviours make existing examples incomplete.
  - Update the **Log files** section if the log format changes.
  - Update **Volume status values** if `checkAccessibility` returns new status strings.

- **`AGENTS.md`** — agent-facing codebase reference.
  - Keep the **CLI flags table** (`Long | Short | Variable | Default`) in sync with `main()`.
  - Update the **Data flow**, **Key design decisions**, or **State map key** sections when the architecture changes.
  - Update the **Log format** section if log output changes.
  - Update the **Extending the project** section when new extension points are added or existing ones change.

Never silently skip these updates. If a change is small (e.g. a default value), a one-line update to the table is sufficient — do not rewrite entire sections unnecessarily.
